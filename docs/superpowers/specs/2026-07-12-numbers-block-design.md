# Дизайн: блок «Числа: считаем по-французски»

Дата: 2026-07-12

## Проблема

До `l09` («Меню и цены») курс не даёт систематического счёта — только `un/une` как артикли и разрозненные числа в диалогах (`l07` — neuf/dix, не объяснённые как система). При этом `l09` уже требует чисел (цены в евро), а полноценный урок чисел (`l14`, «Числа и время: считаем и планируем») стоит намного позже (`module:a1:daily-services`), покрывает только 0-100 и намеренно пропускает нерегулярные 70-99 («за пределами практического A1» — комментарий в `l14-e2.explanation`). Ни один урок не даёт сотни/тысячи/миллионы/миллиарды — то есть уверенного счёта до миллиарда в курсе просто нет.

Формат существующего урока (5-9 слов словаря, 3 упражнения, диалог на 3-4 реплики) не вмещает полную систему счёта — материал по объёму сильно больше обычного урока, но грамматики и произношения в нём меньше.

## Выбранный подход

Решения приняты в диалоге с пользователем перед написанием спеки:

1. **Формат** — мини-модуль из 3 коротких уроков вместо одного раздутого урока: (0-20), (20-100, включая нерегулярные 70-99), (100 и выше — сотни/тысячи/миллионы/миллиарды). Каждый урок остаётся в привычном формате (диалог, упражнения, cards), но со словарём, расширенным под числа.
2. **Место в курсе** — новый модуль `module:starter:numbers`, вставленный перед `module:starter:reception-review` (который содержит `l09`), а не отдельные уроки внутри `reception-review`. Соответствует существующему паттерну маленьких тематических модулей в `starter` (`first-contact`, `transactions`, `orientation-plans`, `help`).
3. **`l14`** — удаляется полностью (дублирует материал нового блока, но с более узким охватом и без сотен/тысяч). Её грамматика `il y a` переносится в `l08a` (первый урок нового блока) — она нужна ровно там, где появляются первые числа для практики.
4. **Прогресс** — новые уроки получают свежие id (`l08a`, `l08b`, `l08c`), никакие существующие id/module/order у уже пройденных уроков (`l01`-`l08`) не меняются. Порядок в приложении определяется полями `order`/`moduleId`, а не числом в id (`app.js:compareLessons`), так что вставка не требует переименований. `currentLessonId` пользователя (`l09`) остаётся валидным — просто у `l09` добавляется дополнительный пререквизит.

Отклонённые варианты:
- Один большой урок — отклонён пользователем: нарушает привычный размер уроков и SRS-нагрузку одной пачкой.
- Правка `l14` вместо удаления — отклонена пользователем в пользу чистого удаления с переносом `il y a`.
- Числа как уроки внутри `reception-review` (без нового модуля) — отклонено пользователем в пользу отдельного модуля, повторяющего существующий паттерн.

## Архитектура

### Новый модуль

```json
{
  "id": "module:starter:numbers",
  "levelId": "starter",
  "title": "Числа: считаем по-французски",
  "description": "Полная система счёта: от нуля до миллиарда.",
  "order": 5,
  "prerequisites": ["module:starter:help"]
}
```

`module:starter:reception-review.order` меняется с `5` на `6`, `prerequisites` остаётся `["module:starter:help"]` → меняется на `["module:starter:numbers"]` (следующий модуль в цепочке).

### Новые уроки

| id | order | title | grammarTopic | pronunciationTopic | prerequisites |
|---|---|---|---|---|---|
| `l08a` | 1 | Числа 0-20: считаем с нуля | `il-y-a` (существующий, переносится из `l14`) | `liaison` | `["l08"]` |
| `l08b` | 2 | Числа 20-100: десятки и особая логика 70-99 | `numbers-tens` (новый) | `nasals` | `["l08a"]` |
| `l08c` | 3 | Числа 100 и больше: сотни, тысячи, миллионы, миллиарды | `numbers-large` (новый) | `silent-endings` | `["l08b"]` |

`l09.prerequisites` меняется с `["l08"]` на `["l08c"]`.

Содержание (объём, не точный текст — формулировки/диалоги/IPA пишутся при наполнении `data/lessons.json`):

- **`l08a`**: числительные 0-20 как отдельные словарные единицы (каждое — своя карточка, чтобы дрилиться по отдельности через SRS). Грамматика `il y a` на этих числах (`il y a trois chaises` / `il y a dix personnes`). Упражнения: перевод фразы с двумя числами, устный счёт/простые суммы в пределах 20, диктант короткой фразы с числом.
- **`l08b`**: trente/quarante/cinquante/soixante как база; отдельно — нерегулярная зона 70-99 (soixante-dix, quatre-vingts, quatre-vingt-dix) и правило `et`/дефис (vingt **et** un, но quatre-vingt-**un** без et; soixante **et** onze). Упражнения: озвучить 71/80/91/99, gap-fill на et/дефис, перевод фраз с возрастом/количеством.
- **`l08c`**: cent/cent un/deux cents vs deux cent trois (согласование `s`), mille (не изменяется, без артикля «un mille»), un million / un milliard как существительные, требующие `de` перед следующим словом (`un million d'habitants`, `un milliard de dollars`). Упражнения: gap-fill на согласование сотен, перевод фраз с населением/ценой, reading-comprehension на коротком тексте с большим числом.

## Последствия для прогресса пользователя

- `user-data/french-study-data.json`: записи `schedules`/`reviewLogs` для карточек `l14` (2 реальных ревью на слове "onze") становятся осиротевшими — приложение просто перестаёт их показывать, ничего не падает (карточки строятся заново из `data/lessons.json` в рантайме, `cards.js:buildCards`). Явно принято пользователем при выборе варианта «удалить `l14` полностью».
- `kv.appState.completedLessons` (`l01`-`l08`) и `currentLessonId` (`l09`) остаются валидными без изменений — новые уроки просто встают в очередь перед `l09` по цепочке пререквизитов.
- Схема `mastery.js` (проверка пререквизитов через closure по `module.prerequisites`/`level.prerequisites`) не меняется — новый модуль и уроки используют тот же формат `prerequisites`, что и остальной курс.

## Аудио

После того как `l08a`/`l08b`/`l08c` дописаны в `data/lessons.json`, прогнать `python3 scripts/prewarm_tts.py`, закоммитить новые `data/audio/*.mp3`/обновлённый `data/audio/manifest.json` — тот же процесс, что при добавлении предыдущих уроков. Файлы озвучены по хэшу текста, а не по уроку (`data/audio/<hash>.mp3`), так что удаление `l14` не оставляет отдельную папку для чистки — фразы, которые больше нигде не встречаются, просто перестанут попадать в манифест при следующем прогоне `prewarm_tts.py`; сами `.mp3`-файлы можно оставить как есть (не мешают) или почистить `git rm` вручную — не блокирует спеку.

- `node tests/exercises.mjs`, `node tests/card-manifest.mjs` — уже читают `data/lessons.json`, должны продолжать проходить без изменений тестового кода.

## Границы (что не входит)

- A2/B1/B2 не затрагиваются.
- Точные тексты диалогов, полные списки словаря с IPA, точные формулировки упражнений — не часть этой спеки, пишутся при наполнении `data/lessons.json` по плану реализации.
- Часы/время (`heure`, «который час») не входят в блок — это отдельная тема, частично уже покрытая `l15`/`l17`; данный блок — только кардинальные числительные.
- Порядковые числительные (premier, deuxième...) не входят — вне запроса пользователя («считать», не «нумеровать»).
