# Дизайн: качественный TTS

Дата: 2026-07-09

## Проблема

Текущая озвучка ([app.js:992-999](../../../app.js#L992)) использует браузерный `speechSynthesis` (Web Speech API): голос берётся из того, что установлено в ОС/браузере пользователя, без какого-либо контроля качества. Итог: звук робото-механический, произношение иногда откровенно неправильное, а картина различается от устройства к устройству. Это касается как фраз/словаря из `data/lessons.json`, так и произвольных слов, которые пользователь добавляет в разделе «Словарь» — оба случая одинаково важны.

Ограничение: проект принципиально локальный и бесплатный («платные курсы и подписки не используются», README). Внешние платные API исключены.

## Выбранный подход

Локальный TTS-прокси на Python поверх `edge-tts` (MIT, открытая библиотека, использующая бесплатные neural-голоса Microsoft Edge Read Aloud — `fr-FR-DeniseNeural`, `fr-FR-HenriNeural`) + кэширование, с разделением на:

- **Контент уроков (`data/lessons.json`)** — озвучивается заранее (build-time), статические mp3 коммитятся в репозиторий и проигрываются напрямую, без обращения к серверу.
- **Произвольный текст (свои слова из «Словаря», либо lesson-фраза с нестандартной скоростью/голосом)** — озвучивается на лету через локальный `/tts`-эндпоинт, результат кэшируется в IndexedDB на стороне браузера.
- **Fallback** — если сервер/сеть/edge-tts недоступны, тихо откатываемся на нынешний `speechSynthesis` для конкретного проигрывания. Приложение никогда не остаётся совсем без звука.

Рассмотренные и отклонённые варианты:
- **Только пре-генерация статики** — не решает вопрос со своими словами (пользователь подтвердил: одинаково важны).
- **Piper через WASM целиком в браузере** — полностью офлайн и без зависимости от Microsoft, но заметно хуже по качеству голоса и сложнее в реализации (воркер, загрузка модели 30-60 МБ). Не выбран, т.к. вариант с edge-tts даёт заметно лучшее качество бесплатно.
- **Прямые unofficial-вызовы из браузера к Google Translate TTS / Edge TTS** — упираются в CORS и отсутствие бэкенда (у проекта его вообще не было), либо требуют подписи токена, которую в чистом клиенте не воспроизвести надёжно. Локальный Python-прокси снимает эту проблему, так как `python3` и так уже требуется для запуска проекта.

## Архитектура

```
Браузер                          server.py (localhost:5173)
┌─────────────────┐              ┌──────────────────────────┐
│ speakFrench(text)│             │ статика (как раньше)      │
│  1. lesson-манифест? → играть  │ GET /tts?text&voice&rate  │
│     data/audio/*.mp3 напрямую  │   → edge-tts synth        │
│  2. IndexedDB ttsAudio кэш?    │   → диск-кэш по хэшу       │
│  3. fetch /tts → кэш + играть  │   → mp3 (Cache-Control     │
│  4. ошибка → speechSynthesis   │      immutable)            │
└─────────────────┘              └──────────────────────────┘
```

## Сервер (`server.py`)

Заменяет `python3 -m http.server 5173` в README (тот же порт, тот же однокомандный запуск).

- `ThreadingHTTPServer` + кастомный handler: отдаёт статику как раньше, плюс роут `/tts`.
- `GET /tts?text=<url-encoded>&voice=fr-FR-DeniseNeural|fr-FR-HenriNeural&rate=<0.55-1.10>` — `voice` передаётся полным neural voice id, тем же, что хранится в `state.settings.voiceURI` на фронтенде (без отдельного маппинга short-name ↔ full-id).
  - нормализация текста (trim, схлопнуть пробелы);
  - `rate` округляется до ближайших 5% и конвертируется в формат edge-tts (`"-18%"` и т.п.);
  - cache key считается общей функцией (см. «Кэш-ключ» ниже);
  - если `<cache-dir>/<key>.mp3` существует — отдать как есть;
  - иначе синтезировать через `edge-tts`, сохранить на диск, отдать;
  - при ошибке (нет пакета `edge-tts`, нет сети, апстрим недоступен) — `502 {"error": "..."}`; фронтенд трактует это как сигнал к fallback.
  - `Cache-Control: public, max-age=31536000, immutable` — контент детерминирован по ключу.
- `edge-tts` — новая опциональная зависимость (`pip install edge-tts`). Без неё сервер продолжает отдавать статику и уже сгенерированные lesson-файлы; `/tts` просто всегда возвращает 502, и всё уходит в браузерный fallback — ничего не ломается.

### Кэш-ключ (общий для сервера, прогрева и фронтенда)

`key = sha256(normalizedText + "|" + voiceId + "|" + roundedRatePercent)`, где:
- `normalizedText` — text.trim() со схлопнутыми пробелами;
- `voiceId` — полный id голоса (`fr-FR-DeniseNeural` / `fr-FR-HenriNeural`);
- `roundedRatePercent` — rate, округлённый до ближайших 5% и записанный как `"-18%"` и т.п.

Это единственное место, определяющее ключ. `server.py` реализует эту функцию, `scripts/prewarm-tts.py` импортирует её из `server.py` (не дублирует), фронтенд реализует эквивалент на JS (`crypto.subtle.digest("SHA-256", ...)`) для двух целей: (1) поиск в `data/audio/manifest.json`, (2) поиск/запись в IndexedDB `ttsAudio`. Ключ в манифесте и в IndexedDB — один и тот же формат, так что при переходе с дефолтного rate/voice на нестандартный запись просто попадает в другую часть кэша без специальной логики "переключения источника".

## Прогрев кэша для уроков (`scripts/prewarm-tts.py`)

- Читает `data/lessons.json`, собирает весь озвучиваемый текст: `targetPhrase`, реплики `dialogue`, `vocabulary[].audioText`/слово, тексты произношения (то, что передаётся в `renderVoiceLab`).
- Для каждого текста синтезирует на **голосе Denise, скорости 0.82** (текущий дефолт), переиспользуя ту же функцию синтеза, что и `/tts` в `server.py` (без дублирования логики).
- Результат: `data/audio/<key>.mp3` + `data/audio/manifest.json` (`{ cacheKey: "filename.mp3" }`, где `cacheKey` — та же функция, что описана в «Кэш-ключ»), оба коммитятся в git.
- Запускается вручную автором курса при добавлении/правке урока — не часть рантайма приложения (аналогично ручному прогону `node tests/*.mjs`).
- Henri заранее не прогревается — переключение на Henri для lesson-текста уходит в live `/tts` + IndexedDB, как и нестандартная скорость.

## Фронтенд

### Кэш в IndexedDB

- Новый store `ttsAudio` в `STORE_NAMES` ([storage.js:4](../../../storage.js#L4)), по образцу `recordings`. Версия БД бампается — существующая логика апгрейда ([storage.js:19-26](../../../storage.js#L19)) создаст store автоматически, миграция данных не нужна.
- Запись: `{ id: cacheKey, blob, text, voice, rate, createdAt }`.

### `speakFrench(text)`

1. Если `text` есть в `data/audio/manifest.json` **и** текущие voice/rate равны дефолтным (Denise, 0.82) — играть статический файл напрямую, в IndexedDB не писать (уже статика).
2. Иначе — посчитать cache key, проверить `ttsAudio` в IndexedDB → если есть, играть blob.
3. Иначе — `fetch('/tts?...')`; при успехе положить blob в кэш и играть.
4. При ошибке fetch/502 — старый путь через `speechSynthesis` (переименован в `speakFrenchFallback`, логика не меняется).

### Настройки ([app.js:395-411](../../../app.js#L395))

- `voice-select` → два фиксированных варианта: **Denise (дефолт)**, Henri. Больше не зависит от голосов ОС.
- Убрать: `refreshVoices`, `voicePriority`, слушатель `voiceschanged`, `state.voices` — не нужны, голоса больше не определяются системой.
- `defaultSettings()` ([app.js:1566](../../../app.js#L1566)): `voiceURI: "fr-FR-DeniseNeural"`.
- Если в сторадже пользователя старое значение `voiceURI` (id системного голоса) — оно не совпадёт ни с одним из двух новых id, при загрузке откатывается на дефолт. Отдельной миграции стораджа не требуется.
- Слайдер скорости остаётся как есть (0.55–1.10, шаг 0.05); значение конвертируется в rate-параметр для `/tts` вместо `utterance.rate`.

## Fallback и обработка ошибок

- `speakFrenchFallback` (бывший `speakFrench`) не меняется по логике — вызывается при любой неудаче основного пути (нет `/tts`, нет сети, edge-tts недоступен).
- Ошибки не показываются пользователю модалкой — тихий fallback + `console.warn` для отладки.

## Документация

- `README.md`: шаг запуска меняется на `python3 server.py` (тот же порт); `pip install edge-tts` описывается как опциональный шаг («без него всё работает, но свои слова озвучиваются через старый браузерный голос»); добавляется описание `scripts/prewarm-tts.py`.

## Тесты

- Новый `tests/tts-cache.mjs` (либо расширение `tests/technical.mjs`): проверка вычисления cache key (нормализация текста, округление rate до 5%), проверка соответствия `data/audio/manifest.json` реальным файлам на диске.
- Серверная часть (`server.py`) не покрывается юнит-тестами на реальный edge-tts (требует сети); роутинг `/tts` и дисковый кэш-хит можно проверить с замоканным HTTP-слоем.

## Границы (что не входит)

- Не меняется UX кнопок "прослушать" — интерфейс тот же, меняется только то, что происходит по клику.
- Не меняется формат `data/lessons.json`.
- Не вводится миграция старых значений `voiceURI` в сторадже — просто откат на дефолт при несовпадении.
- Голоса ограничены двумя вариантами (Denise/Henri); добавление других голосов — вне рамок этой спеки.
